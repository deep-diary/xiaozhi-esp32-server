# DeepDiary 系统概述

## 一、项目介绍

### 1.1 项目定位

**DeepDiary** 是一个智能记忆与资源追溯系统，旨在帮助用户追溯记忆、管理资源、创造价值。

**系统组成：**

- **ESP32 AI 设备**：系统入口，负责感知环境、采集数据、人机交互
- **xiaozhi-server**：系统大脑，负责数据处理、智能分析、决策推荐

### 1.2 核心价值

1. **记忆追溯**：帮助用户追溯和回忆重要记忆，增强记忆能力
2. **资源追溯**：帮助用户发现和利用资源，创造价值
3. **智能交互**：通过自然语言交互，降低使用门槛
4. **自动化处理**：自动识别、记录、匹配，减少人工操作

### 1.3 核心功能

#### 1.3.1 记忆追溯系统（Memory Tracing）

基于多源数据（日记、照片、GPS 轨迹、聊天记录）实现智能记忆追溯，支持时间线、人物、地点、事件等多维度检索。

#### 1.3.2 资源追溯系统（Resource Tracing）

基于三大资源和三大需求，实现智能匹配和机会推荐，帮助用户发现服务机会，实现资源变现。

#### 1.3.3 智能交互系统

通过 ESP32 设备和 Web 界面，实现语音、文字、图片等多种交互方式，提供自然流畅的用户体验。

## 二、应用场景

### 2.1 场景一：主动查询照片

用户通过语音或文字查询历史照片，系统自动检索相关记忆并展示。

**示例：** "我想看我跟张三在哈尔滨一起滑雪的照片"

### 2.2 场景二：自动识别与主动交互

系统自动识别来访者，检索相关记忆，主动发起对话。

**示例：** 朋友张三来访，系统识别后主动聊天："上次你们一起去了哈尔滨滑雪，玩得怎么样？"

### 2.3 场景三：新朋友识别与资源追溯

系统自动识别新朋友，引导录入信息，匹配资源和需求。

**示例：** 新朋友来访 → 自动识别 → 引导录入资源需求 → 智能匹配 → 推荐服务机会

### 2.4 场景四：智能家庭助手

在家庭环境中，管理家庭记忆，优化家庭资源利用。

### 2.5 场景五：智能办公助手

在办公环境中，识别客户，记录会议，匹配业务资源。

### 2.6 场景六：智能社区服务平台

在社区环境中，连接居民，匹配社区资源，促进邻里互助。

## 三、技术架构

### 3.1 系统架构

```
ESP32 AI设备 (入口)
    ↓
xiaozhi-server (大脑)
    ├─ 记忆追溯服务
    ├─ 资源追溯服务
    ├─ Web界面服务
    └─ 设备管理服务
```

### 3.2 核心技术

- **WebSocket**：实时双向通信
- **向量数据库**：语义搜索和匹配
- **LLM**：自然语言理解和生成
- **人脸识别**：Immich 服务集成
- **声纹识别**：声纹服务集成
