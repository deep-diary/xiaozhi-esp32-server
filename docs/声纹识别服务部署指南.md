# 声纹识别服务部署指南

## 1. 文档概述

### 1.1 编写目的

本文档详细说明如何在 DeepDiary 项目中部署和配置声纹识别服务，包括数据库配置、服务部署、接口集成和测试验证等完整流程。

### 1.2 适用范围

- 适用于使用 `docker-compose_all.yml` 部署的小智服务器环境
- 支持全模块部署和最简化部署两种模式
- 需要 MySQL 数据库支持

### 1.3 前置条件

- 已部署小智服务器（xiaozhi-esp32-server）
- MySQL 数据库已运行并可访问
- Docker 和 Docker Compose 已安装
- 宿主机可访问 MySQL 的 3306 端口

### 1.4 快速开始

**推荐方式：使用自动化脚本（最简单）**

```bash
# 进入项目根目录
cd /Users/hanli/Desktop/DeepDiary

# 执行自动化脚本（会自动检查 MySQL 服务并创建数据库）
./script/setup-voiceprint-db.sh
```

脚本会自动完成以下操作：
- ✅ 检查 Docker 是否安装
- ✅ 检查 MySQL 容器是否运行
- ✅ 检查容器健康状态
- ✅ 检查 MySQL 服务连接
- ✅ 执行 SQL 脚本创建数据库和表
- ✅ 验证创建结果

**手动方式：直接执行 SQL 脚本**

如果你熟悉 MySQL 操作，也可以手动执行：

```bash
# 进入脚本目录
cd /Users/hanli/Desktop/DeepDiary/script

# 执行 SQL 脚本
docker exec -i xiaozhi-esp32-server-db mysql -u root -p123456 < create_voiceprint_db.sql
```

**注意**：请根据实际情况修改密码和路径。

## 2. 环境检查

### 2.1 检查 MySQL 服务状态

首先检查 MySQL 容器是否正常运行：

```bash
docker ps --filter "name=xiaozhi-esp32-server-db"
```

预期输出应显示容器状态为 `Up` 且健康检查通过。

### 2.2 检查 MySQL 端口映射

确认 `docker-compose_all.yml` 中 MySQL 服务已正确配置端口映射：

```yaml
xiaozhi-esp32-server-db:
  ...
  ports:
    - "3306:3306"  # 确保使用 ports 而不是 expose
```

**注意**：如果使用的是 `expose` 而不是 `ports`，需要修改配置并重启容器。

### 2.3 测试 MySQL 连接

使用以下命令测试 MySQL 端口是否可访问：

```bash
# 方法1：使用 nc 命令（如果已安装）
nc -zv 127.0.0.1 3306

# 方法2：使用 MySQL 客户端直接连接
mysql -h 127.0.0.1 -P 3306 -u root -p
```

如果连接失败，请检查：
- Docker 容器是否正常运行
- 防火墙是否阻止了 3306 端口
- `docker-compose_all.yml` 中的端口配置是否正确

## 3. 数据库配置

### 3.1 连接到 MySQL 数据库

在执行 SQL 语句之前，需要先连接到 MySQL 数据库。根据你的环境，有以下几种连接方式：

#### 方法一：通过 Docker 容器连接（推荐）

如果你的 MySQL 是通过 Docker 部署的，可以直接在容器内执行 SQL 命令：

```bash
# 方式1：交互式连接（推荐，适合手动执行 SQL）
docker exec -it xiaozhi-esp32-server-db mysql -u root -p123456

# 连接成功后，你会看到 MySQL 提示符：mysql>
# 然后就可以直接输入 SQL 语句了
```

**执行步骤**：
1. 运行上述命令后，会进入 MySQL 命令行界面
2. 你会看到 `mysql>` 提示符
3. 直接输入 SQL 语句，按回车执行
4. 输入 `exit;` 或按 `Ctrl+D` 退出

**示例**：
```bash
$ docker exec -it xiaozhi-esp32-server-db mysql -u root -p123456
Welcome to the MySQL monitor.  Commands end with ; or \g.
...
mysql> SHOW DATABASES;
mysql> CREATE DATABASE voiceprint_db;
mysql> exit;
Bye
```

#### 方法二：通过本地 MySQL 客户端连接

如果你在宿主机上安装了 MySQL 客户端，可以通过以下方式连接：

```bash
# 连接到 MySQL（需要输入密码）
mysql -h 127.0.0.1 -P 3306 -u root -p

# 或者如果密码是 123456，可以直接指定
mysql -h 127.0.0.1 -P 3306 -u root -p123456
```

**参数说明**：
- `-h 127.0.0.1`：MySQL 服务器地址（本地）
- `-P 3306`：MySQL 端口
- `-u root`：用户名
- `-p`：提示输入密码（或 `-p123456` 直接指定密码）

#### 方法三：通过 SQL 文件执行（推荐，适合批量执行）

**方式1：使用自动化脚本（最推荐）**

项目 `script` 目录下提供了自动化脚本 `setup-voiceprint-db.sh`，会自动检查 MySQL 服务并执行 SQL：

```bash
# 进入项目根目录
cd /Users/hanli/Desktop/DeepDiary

# 执行自动化脚本
./script/setup-voiceprint-db.sh
```

**方式2：直接执行 SQL 文件**

项目 `script` 目录下已提供了 `create_voiceprint_db.sql` 脚本文件，可以直接使用：

```bash
# 进入脚本目录
cd /Users/hanli/Desktop/DeepDiary/script

# 执行 SQL 脚本
docker exec -i xiaozhi-esp32-server-db mysql -u root -p123456 < create_voiceprint_db.sql
```

**或者手动创建 SQL 文件**：

```bash
# 创建 SQL 文件
cat > create_voiceprint_db.sql << 'EOF'
CREATE DATABASE IF NOT EXISTS voiceprint_db 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

USE voiceprint_db;

CREATE TABLE IF NOT EXISTS voiceprints (
    id INT AUTO_INCREMENT PRIMARY KEY,
    speaker_id VARCHAR(255) NOT NULL UNIQUE COMMENT '说话人ID',
    feature_vector LONGBLOB NOT NULL COMMENT '声纹特征向量',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_speaker_id (speaker_id)
) ENGINE=InnoDB 
DEFAULT CHARSET=utf8mb4 
COLLATE=utf8mb4_unicode_ci 
COMMENT='声纹识别数据表';
EOF

# 执行 SQL 文件
docker exec -i xiaozhi-esp32-server-db mysql -u root -p123456 < create_voiceprint_db.sql
```

#### 方法四：一行命令执行 SQL（适合简单操作）

对于单个 SQL 语句，可以直接通过命令行执行：

```bash
# 执行单个 SQL 命令
docker exec xiaozhi-esp32-server-db mysql -u root -p123456 -e "CREATE DATABASE IF NOT EXISTS voiceprint_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"

# 执行多个 SQL 命令（用分号分隔）
docker exec xiaozhi-esp32-server-db mysql -u root -p123456 -e "USE voiceprint_db; SHOW TABLES;"
```

**注意事项**：
- 如果密码包含特殊字符，需要用引号包裹
- SQL 语句末尾的分号 `;` 是可选的（在 `-e` 模式下）
- 使用 `-i` 参数可以保持交互式输入

### 3.2 创建声纹识别数据库

## 4. 声纹识别服务部署

### 4.1 获取项目源码

**方法一：从 GitHub 下载**

1. 访问 [声纹识别项目地址](https://github.com/xinnan-tech/voiceprint-api)
2. 点击 `Code` 按钮，选择 `Download ZIP`
3. 解压下载的文件
4. 将解压后的文件夹重命名为 `voiceprint-api`

**方法二：使用 Git 克隆（推荐）**

```bash
cd /path/to/your/project
git clone https://github.com/xinnan-tech/voiceprint-api.git
cd voiceprint-api
```

### 4.2 配置数据库连接

#### 4.2.1 创建配置目录

```bash
cd voiceprint-api
mkdir -p data
```

#### 4.2.2 复制配置文件

```bash
# 复制配置文件模板
cp voiceprint.yaml data/.voiceprint.yaml
```

#### 4.2.3 编辑配置文件

编辑 `data/.voiceprint.yaml` 文件，配置数据库连接信息：

```yaml
server:
  # 服务监听地址
  host: 0.0.0.0
  # 服务监听端口
  port: 8005
  # 接口访问令牌（如果为空会自动生成）
  authorization: 

mysql:
  # MySQL 数据库主机地址
  # 注意：如果使用 Docker 部署，需要填写宿主机的局域网 IP 地址
  # 例如：192.168.1.25，而不是 127.0.0.1
  host: "192.168.1.25"  # 替换为你的宿主机 IP
  # 端口
  port: 3306
  # 用户名
  user: "root"
  # 用户密码（根据你的实际密码修改）
  password: "123456"
  # 数据库名
  database: "voiceprint_db"
```

**重要提示**：
- 如果声纹识别服务使用 Docker 部署，`host` 必须填写宿主机的**局域网 IP 地址**，不能使用 `127.0.0.1`
- 可以通过以下命令获取本机 IP 地址：
  ```bash
  # macOS/Linux
  ifconfig | grep "inet " | grep -v 127.0.0.1
  
  # 或者
  ip addr show | grep "inet " | grep -v 127.0.0.1
  ```

### 4.3 启动服务

#### 4.3.1 使用 Docker Compose 部署（推荐）

```bash
# 进入项目根目录
cd voiceprint-api

# 停止并删除旧容器（如果存在）
docker compose -f docker-compose.yml down
docker stop voiceprint-api 2>/dev/null || true
docker rm voiceprint-api 2>/dev/null || true

# 启动服务
docker compose -f docker-compose.yml up -d

# 查看日志
docker logs -f voiceprint-api
```

#### 4.3.2 验证服务启动

查看日志，确认服务正常启动。正常启动后，日志会显示类似以下内容：

```
250711 INFO-🚀 开始: 生产环境服务启动（Uvicorn），监听地址: 0.0.0.0:8005
250711 INFO-============================================================
250711 INFO-声纹接口地址: http://127.0.0.1:8005/voiceprint/health?key=abcd
250711 INFO-============================================================
```

**重要**：记录日志中显示的 `key` 值，后续配置需要使用。

### 4.4 获取声纹接口地址

从日志中获取声纹接口地址，例如：
```
http://127.0.0.1:8005/voiceprint/health?key=abcd
```

**由于使用 Docker 部署，需要将地址中的 `127.0.0.1` 替换为宿主机的局域网 IP**：

```bash
# 原地址
http://127.0.0.1:8005/voiceprint/health?key=abcd

# 修改后（假设宿主机 IP 为 192.168.1.25）
http://192.168.1.25:8005/voiceprint/health?key=abcd
```

### 4.5 测试接口

在浏览器中访问修改后的声纹接口地址，应该返回类似以下内容：

```json
{
  "total_voiceprints": 0,
  "status": "healthy"
}
```

如果返回上述内容，说明声纹识别服务部署成功。

## 5. 集成配置

### 5.1 全模块部署配置

#### 5.1.1 在智控台配置接口

1. 使用管理员账号登录智控台
2. 点击顶部 `参数字典`，选择 `参数管理` 功能
3. 搜索参数 `server.voice_print`
4. 点击修改按钮，将上一步获取的**声纹接口地址**（已替换为局域网 IP）粘贴到 `参数值` 中
5. 保存配置

#### 5.1.2 配置智能体记忆模式

1. 进入智能体的角色配置页面
2. 将记忆设置改为 `本地短期记忆`
3. **重要**：确保开启 `上报文字+语音` 选项
4. 保存配置

#### 5.1.3 注册声纹

1. 将设备通电，与智能体进行正常对话（使用正常语速和音调）
2. 在智控台的 `智能体管理` 页面
3. 找到对应智能体的面板，点击 `声纹识别` 按钮
4. 点击底部的 `新增` 按钮
5. 在弹出的对话框中：
   - 选择要注册的语音片段
   - 填写 `描述`（建议填写说话人的职业、性格、爱好等信息，方便智能体识别和分析）
   - 保存

#### 5.1.4 测试声纹识别

1. 与智能体进行对话
2. 询问："你知道我是谁吗？"
3. 如果智能体能够正确识别说话人，说明声纹识别功能正常工作

### 5.2 最简化部署配置

#### 5.2.1 配置文件位置

编辑 `xiaozhi-server/data/.config.yaml` 文件（如果不存在需要创建）。

#### 5.2.2 添加声纹识别配置

在配置文件中添加以下内容：

```yaml
# 声纹识别配置
voiceprint:
  # 声纹接口地址（使用局域网 IP）
  url: http://192.168.1.25:8005/voiceprint/health?key=abcd
  
  # 说话人配置：speaker_id,名称,描述
  speakers:
    - "test1,张三,张三是一个程序员"
    - "test2,李四,李四是一个产品经理"
    - "test3,王五,王五是一个设计师"
```

**配置说明**：
- `url`：填写上一步获取的声纹接口地址（已替换为局域网 IP）
- `speakers`：配置说话人信息，格式为 `speaker_id,名称,描述`
  - `speaker_id`：唯一标识符，后续注册声纹时会用到
  - 名称：说话人的显示名称
  - 描述：说话人的详细信息

#### 5.2.3 注册声纹

##### 方法一：使用 API 注册

1. 访问 API 文档：`http://localhost:8005/voiceprint/docs`

2. 获取访问令牌（Token）：
   - Token 为声纹接口地址中 `?key=` 后的部分
   - 例如：地址为 `http://192.168.1.25:8005/voiceprint/health?key=abcd`，则 Token 为 `abcd`

3. 使用 curl 注册声纹：

```bash
curl -X POST \
  -H "Authorization: Bearer abcd" \
  -F "speaker_id=test1" \
  -F "file=@/path/to/audio.wav" \
  http://192.168.1.25:8005/voiceprint/register
```

**参数说明**：
- `speaker_id`：必须与 `.config.yaml` 中配置的 `speaker_id` 一致
- `file`：要注册的音频文件路径（WAV 格式）
- `Authorization`：Bearer Token，从接口地址中提取

##### 方法二：使用 Swagger UI 注册

1. 访问 `http://localhost:8005/voiceprint/docs`
2. 找到 `POST /voiceprint/register` 接口
3. 点击 `Try it out`
4. 填写参数：
   - `speaker_id`：例如 `test1`
   - `file`：上传音频文件
5. 在 `Authorization` 中输入 Token（格式：`Bearer abcd`）
6. 点击 `Execute` 执行

#### 5.2.4 启动服务

```bash
# 启动小智服务器
cd xiaozhi-server
docker compose -f docker-compose_all.yml up -d

# 确保声纹识别服务也在运行
cd voiceprint-api
docker compose -f docker-compose.yml up -d
```

## 6. 故障排查

### 6.1 MySQL 连接问题

**问题**：无法连接到 MySQL 数据库

**排查步骤**：
1. 检查 MySQL 容器是否运行：
   ```bash
   docker ps | grep xiaozhi-esp32-server-db
   ```

2. 检查端口映射：
   ```bash
   docker port xiaozhi-esp32-server-db
   ```
   应该显示 `3306/tcp -> 0.0.0.0:3306`

3. 测试端口连通性：
   ```bash
   nc -zv 127.0.0.1 3306
   ```

4. 检查防火墙设置

5. 验证数据库配置信息（用户名、密码、数据库名）

### 6.2 声纹服务无法访问

**问题**：无法访问声纹识别接口

**排查步骤**：
1. 检查服务是否运行：
   ```bash
   docker ps | grep voiceprint-api
   docker logs voiceprint-api
   ```

2. 检查端口是否映射：
   ```bash
   docker port voiceprint-api
   ```

3. 确认使用的是局域网 IP 而不是 `127.0.0.1`

4. 检查防火墙是否允许 8005 端口

### 6.3 声纹识别不工作

**问题**：配置后声纹识别功能不生效

**排查步骤**：
1. 确认智能体记忆模式设置为 `本地短期记忆`
2. 确认已开启 `上报文字+语音` 选项
3. 检查声纹接口地址配置是否正确（使用局域网 IP）
4. 验证声纹是否已成功注册：
   ```bash
   # 访问健康检查接口
   curl "http://192.168.1.25:8005/voiceprint/health?key=abcd"
   ```
   查看返回的 `total_voiceprints` 数量

5. 检查数据库中的声纹记录：
   ```sql
   USE voiceprint_db;
   SELECT speaker_id, created_at FROM voiceprints;
   ```

### 6.4 常见错误信息

| 错误信息 | 可能原因 | 解决方法 |
|---------|---------|---------|
| `Connection refused` | MySQL 端口未映射或服务未启动 | 检查 docker-compose 配置，重启 MySQL 容器 |
| `Access denied` | 数据库用户名或密码错误 | 检查配置文件中的数据库凭证 |
| `Unknown database` | 数据库不存在 | 执行数据库创建 SQL 语句 |
| `401 Unauthorized` | Token 错误或缺失 | 检查接口地址中的 key 参数 |
| `Network unreachable` | 使用了错误的 IP 地址 | 使用局域网 IP 而不是 127.0.0.1 |
